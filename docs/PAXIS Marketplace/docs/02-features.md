## 核心功能模組

### 1. 訂單管理

- 定時從各電商平台拉取新訂單（Polling）
- 訂單彙整顯示，統一格式
- 訂單狀態追蹤（待出貨 / 已出貨 / 完成 / 退換貨中）
- 訂單篩選、搜尋（依平台、日期、狀態、消費者）
- 批次處理（多筆訂單同時安排出貨）
- 平台出貨時限警示（超時前提醒）

### 2. 消費者資料管理

存放出貨與售後服務必要資料：

```
消費者基本資料：
  - 姓名
  - 電話
  - Email
  - 收件地址

歷史記錄：
  - 歷史訂單列表
  - 退換貨記錄
  - 備註
```

> 注意：不做完整 CRM，只存履行和售後必要欄位。

### 3. 出貨管理

- 選擇物流商、填寫出貨資訊
- 列印出貨標籤（串接各物流商標籤 API）
- 批次出貨（多筆訂單同時出貨）
- 物流追蹤號自動回寫電商平台
- 出貨異常通知（物流商回報異常時通知賣家）

### 4. 履行模式調度

每張訂單可指定履行模式：

```
自有庫存出貨
  └─ （可選）向 Paxis.tw 查詢庫存，確認後扣減

調貨後出貨
  └─ 建立採購需求，待調貨到位後出貨
  └─ 系統追蹤調貨狀態，到貨提醒

供應商直發
  └─ 通知供應商出貨資訊
  └─ 取得追蹤號後回寫平台
```

### 5. 退換貨管理

Follow 各平台既有退換貨機制：

```
退貨流程：
  平台通知退貨申請
    → 賣家確認
    → 消費者寄回
    → 入庫確認
    → 退款（平台處理）
    → 庫存回補通知（若有對接 Paxis）

換貨流程：
  平台通知換貨申請
    → 賣家確認
    → 消費者寄回
    → 入庫確認
    → 重新安排出貨（走正常出貨流程）
```

### 6. 金流對帳

- 各平台撥款記錄彙整
- 退款扣減追蹤
- 各平台待撥款金額顯示
- 匯出對帳報表（CSV）

> 注意：只提供對帳數字，完整財務會計交給客戶的 ERP 或會計軟體。

- 深度庫存管理（交給 Paxis.tw）
- 財務會計、發票開立
