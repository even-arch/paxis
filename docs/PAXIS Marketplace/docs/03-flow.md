# 03 — 核心業務流程

## 主流程：訂單從進來到出貨

```
電商平台（蝦皮 / momo / PChome / 露天 / 酷澎）
  │
  │ 定時 Polling 或 Webhook
  ▼
┌─────────────────────────────────┐
│  訂單進入 Pacture               │
│  - 標準化格式                   │
│  - 建立消費者資料（若為新客）    │
│  - 狀態：待處理                 │
└────────────────┬────────────────┘
                 │
                 ▼
┌─────────────────────────────────┐
│  判斷履行模式                   │
│                                 │
│  A. 自有庫存出貨                │
│  B. 調貨後出貨                  │
│  C. 供應商直發                  │
└────────┬────────────────────────┘
         │
         ├─── A. 自有庫存出貨 ──────────────────────┐
         │                                          │
         │                                          ▼
         │                              ┌───────────────────────┐
         │                              │  選擇物流商 + 填出貨資訊 │
         │                              │  列印出貨標籤           │
         │                              │  實體出貨               │
         │                              └───────────┬───────────┘
         │                                          │
         ├─── B. 調貨後出貨 ──────┐                 │
         │                       │                 │
         │                       ▼                 │
         │           ┌───────────────────┐         │
         │           │  建立採購需求      │         │
         │           │  等待調貨到位      │         │
         │           │  到貨後接上述流程  │         │
         │           └───────────────────┘         │
         │                                         │
         └─── C. 供應商直發 ──────┐                │
                                 │                │
                                 ▼                │
                     ┌───────────────────┐        │
                     │  通知供應商        │        │
                     │  （Email / API）   │        │
                     │  取得追蹤號        │        │
                     └─────────┬─────────┘        │
                               │                  │
                               └──────────────────┘
                                        │
                                        ▼
                           ┌────────────────────────┐
                           │  追蹤號回寫電商平台      │
                           │  訂單狀態更新：已出貨    │
                           └────────────────────────┘
```

## 退貨流程

```
消費者在電商平台申請退貨
  │
  │ 平台通知 Pacture（Webhook 或 Polling）
  ▼
Pacture 顯示退貨申請
  │
  ▼
賣家確認退貨申請
  │
  ▼
消費者寄回商品
  │
  ▼
賣家確認入庫
  │
  ├─ 商品狀況正常 → 庫存回補（通知 Paxis，若有對接）
  │
  └─ 商品損壞 → 記錄備註，不回補庫存
  │
  ▼
退款由電商平台處理（Pacture 不介入金流）
  │
  ▼
更新訂單狀態：退貨完成
```

---

## 換貨流程

```
消費者在電商平台申請換貨
  │
  ▼
賣家確認換貨申請（確認換貨品項庫存）
  │
  ▼
消費者寄回原商品
  │
  ▼
賣家確認收到原商品
  │
  ▼
建立新出貨單（同消費者地址，換貨品項）
  │
  ▼
走正常出貨流程
  │
  ▼
更新訂單狀態：換貨完成
```

---

## 平台出貨時限管理

各平台對出貨時間有明確規定，超時會影響賣家評分或被罰：

| 平台 | 一般出貨時限 | 說明 |
|------|------------|------|
| 蝦皮 | 2 個工作天 | 超時訂單自動取消風險 |
| momo | 依商品設定 | 通常 1-3 天 |
| PChome | 24h 館別有特別要求 | 依商品類型不同 |
| 露天 | 3-5 天 | 相對寬鬆 |
| 酷澎 | 依商品設定 | 台灣版持續調整中 |

**系統處理方式：**
- 訂單進來時，依平台設定計算出貨截止時間
- 截止前 N 小時（可設定）發出警示通知
- 逾期未出貨的訂單標示紅色警示

## 與 Paxis.tw 的對接流程（選配）

```
Pacture 收到訂單
  │
  ▼
向 Paxis API 查詢品項可用庫存
  │
  ├─ 庫存足夠 → 預留庫存（Reservation）→ 安排出貨
  │
  └─ 庫存不足 → 提示賣家 → 賣家決定是否調貨
  │
  ▼
出貨完成後
  │
  ▼
通知 Paxis 扣減庫存（確認出貨）

退貨入庫後
  │
  ▼
通知 Paxis 回補庫存
```
